                Альтернативная история для менеджера FAR
                ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
                        Техническая информация
                        ~~~~~~~~~~~~~~~~~~~~~~

    В данном документе описаны настройки плагина, недоступные для управления
из интерфейса самого плагина.

TechInfo #0
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "MultiplyAddEnabled":REG_DWORD
   Параметр "MultiplyAddEnabled" с типом DWORD управляет поведением плагина
   при обнаружении им множественного добавления в историю FAR (множественное
   добавление - это добавление более одного нового элемента в историю с
   интервалом между появлениями элементов менее 200 миллисекунд).
   Может быть одним из следующих значений:
     0 - В историю AltHistory будет добавлен только последний элемент из
         такого множества.
     1 - В историю AltHistory будут добавлены все элементы из множества, но
         с некоторыми замечаниями:
         а) Эта установка не будет иметь значения (всегда будет добавляться
            только последний элемент), если существует еще один или более
            экземпляров FAR, плагин AltHistory в которых работает с теми же
            файлами истории (см. настройку "Место хранения файлов истории"
            в конфигурации плагина).
         б) Внимание!!! При данной установке и при запуске тем же
            пользователем нового экземпляра FAR без плагина AltHistory,
            плагин в текущем экземпляре может работать некорректно, ложно
            добавляя в историю ранее добавленные элементы.
   По умолчанию значение = 0 (добавляется только последний элемент из
   множества).


TechInfo #1
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "AepApplyNowDefault":REG_DWORD
   Параметр "AepApplyNowDefault" с типом DWORD управляет состоянием флажка
   "Применить к существующей истории" при открытии окна
   "Добавить в список исключений".
   Может быть одним из следующих значений:
     0 - Флажок снят.
     1 - Флажок установлен.
   По умолчанию значение = 1 (Флажок установлен).


TechInfo #2
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "HiddenCommandsSpacesCount":REG_DWORD
   Параметр "HiddenCommandsSpacesCount" с типом DWORD задает количество
   пробелов, которые должны предшествовать команде, попадающей в стандартную
   историю FAR, чтобы она не попала в историю команд AltHistory.
   Минимальное значение = 1, максимальное = 16.
   По умолчанию значение = 4.


TechInfo #3
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "TryExpandCmd":REG_DWORD
   Параметр "TryExpandCmd" с типом DWORD управляет возможностью дописывать
   к командам, сохраняемым в истории команд, полный путь до этих команд.
   Может быть одним из следующих значений:
     0 - Команды сохраняются в истории команд в неизменном виде.
     1 - Если команда представляет собой имя файла (возможно, без расширения,
         если это исполнимый файл), то делается попытка найти этот файл
         относительно текущей папки. Если файл найден - его полное имя
         заменяет собой исходную команду.
   По умолчанию значение = 0 (команды сохраняются в истории команд в
   неизменном виде).


TechInfo #4
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "MatchSelectedColor":REG_DWORD
   "MatchRegularColor":REG_DWORD
   Параметры "MatchSelectedColor" и "MatchRegularColor" с типом DWORD
   определяют цветовой аттрибут символов (кроме символа горячей клавиши)
   в элементе истории, удовлетворяющих текущему фильтру (случаи для
   выделенного элемента и прочих, соответственно).
   Могут быть одним из следующих значений:
     4294967295 (0xFFFFFFFF) - светло-зеленый цвет символов на текущем
          цвете фона.
     0-255 (0-0xFF) - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = 4294967295 (светло-зеленый цвет символов на
   текущем цвете фона).


   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "MatchSelectedHighlightColor":REG_DWORD
   "MatchRegularHighlightColor":REG_DWORD
   Параметры "MatchSelectedHighlightColor" и "MatchRegularHighlightColor" с
   типом DWORD определяют цветовой аттрибут символа горячей клавиши в
   элементе истории, если символ удовлетворяет текущему фильтру (случаи для
   выделенного элемента и прочих, соответственно).
   Могут быть одним из следующих значений:
     4294967295 (0xFFFFFFFF) - цвет символа не меняется (его определяет FAR),
          цвет фона становится равным цвету фона, указанному в
          "MatchSelectedColor" или "MatchRegularColor" (в зависимости от типа
          элемента)
     0-255 (0-0xFF) - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = 4294967295.


TechInfo #5
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "CmdsHistoryLimitPolicy":REG_DWORD
   "ViewHistoryLimitPolicy":REG_DWORD
   "FoldersHistoryLimitPolicy":REG_DWORD
   Эти параметры с типом DWORD управляют возможностью проводить автоматическое
   обновление соответствующих историй при записи их на диск.
   Могут принимать одно из следующих значений:
     0 - Автоматическое обновление не производится.
     1 - Производится автоматическое обновление аналогичное команде F5.
   Для "CmdsHistoryLimitPolicy" доступно еще одно значение:
     2 - Производится автоматическое обновление аналогичное команде Shift-F5.

   По умолчанию значение = 0 (автоматическое обновление не производится).

   Примечание: обновление проходит со следующими отличиями от оригинальных
   команд:
     1. Обновление происходит только если размер истории больше допустимого
        (указанного в диалоге конфигурации).
     2. Обновление прерывается, если размер обновляемой истории становится
        равным допустимому.
     3. При обновлении рассматриваются только пути, расположенные на локальных
        жестких дисках.

TechInfo #9
~~~~~~~~~~~
   [HKEY_CURRENT_USER\Software\Far\Plugins\SNSoft\AltHistory]
   "BackSpaceRemoveFilterWrongPart":REG_DWORD
   Параметр "BackSpaceRemoveFilterWrongPart" с типом DWORD управляет
   поведением плагина при нажатии BackSpace в окне истории.
   Может быть одним из следующих значений:
     0 - BackSpace всегда удаляет последний символ.
     1 - BackSpace удаляет неверную часть фильтра, если она присутствует,
         иначе удаляет последний символ.
   По умолчанию значение = 1 (BackSpace удаляет неверную часть фильтра, если
   она присутствует, иначе удаляет последний символ).
